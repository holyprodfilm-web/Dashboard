# АРМ мониторинга строительства объектов теплоснабжения МО

Веб-приложение для учёта протокольных поручений, совещаний и справочника объектов государственной программы.

## Стек

- React 19 + TypeScript
- Vite
- Tailwind CSS
- Supabase (Auth + PostgreSQL)

## Быстрый старт

```bash
# Установка зависимостей
npm install

# Настройка окружения
cp .env.example .env
# Заполните VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY

# Запуск dev-сервера
npm run dev

# Сборка
npm run build
```

## Настройка базы данных

1. Создайте проект в [Supabase](https://supabase.com)
2. Выполните SQL из `supabase/schema.sql` в SQL Editor
3. Загрузите справочник объектов в таблицу `addresses`
4. Первому пользователю назначьте роль `admin` в таблице `profiles`

## Роли пользователей

| Роль | Права |
|------|-------|
| **admin** | Полный доступ, управление пользователями, удаление |
| **manager** | Создание совещаний и поручений, редактирование своих протоколов |
| **contractor** | Изменение статуса своих поручений |
| **guest** | Только просмотр |

## Структура проекта

```
src/
├── components/     # UI-компоненты
├── lib/            # Supabase, auth, права доступа
├── types.ts        # TypeScript-типы
└── utils.ts        # Вспомогательные функции
```

## Разделы приложения

- **Протокольные поручения** — дашборд, архив совещаний, детали протокола
- **Объекты ГП** — справочник объектов с поиском
- **Пользователи** (только admin) — назначение ролей
